Manual

Web Antenna 取扱説明書

Web Antenna は、RSSがないWebサイトも含めてページ更新を監視する、ブラウザだけで動く更新確認ツールです。登録サイトを確認し、前回取得内容との差分、タグ、ピン、画像一覧、Cloud同期を使って更新チェックを整理できます。

起動antenna.html をブラウザで開くだけで使えます。
保存標準は localStorage。設定すると Supabase に同期します。
スマホ一覧と詳細を切り替える表示。import/export とタイル表示は非表示です。

最短セットアップ

  1. output/antenna/antenna.html をブラウザで開きます。
  2. ヘッダーのURL欄に監視したいWebサイトを入力し、add を押します。https:// は省略できます。
  3. 追加直後に初回確認が始まり、ページ内容のベースラインが保存されます。
  4. 以後は Web Antenna ⟳ ロゴ、または r で全サイトをまとめて確認します。
Cloud同期を使う場合は、先に supabase-schema.sql をSupabase SQL Editorで実行してから、設定画面にSupabase URL / anon key / 共有IDを入力します。

更新確認

全サイトを確認

ヘッダー左上の Web Antenna ⟳ ロゴ、または r を押します。最大3件を並列で確認しながら順次処理します。

1件だけ確認

各行の ⟳、または詳細画面の確認ボタンを押します。対象サイトだけを再取得します。

表示意味
未確認まだ一度も確認していません。
⟳ スピナー確認処理中です。
エラー取得に失敗しました。詳細にエラー理由が表示されます。
バッジなし確認済みで、現在は変更なしです。

差分表示

Web Antenna は取得したページ本文を前回内容と比較し、変化があった場合に差分スニペットを残します。既読にしても次回「変化なし」と確認されるまでは前回差分を参照できます。

表示意味
緑の +今回新たに追加された行です。
赤の −前回から削除された行です。
グレー前後の文脈行です。
· · ·変化のない行を省略しています。

インライン差分: 追加行と削除行がペアになる場合、変更された文字だけをさらに強調します。長い行はクリックで全文表示 / 折りたたみできます。

監視設定

サイトごとに「監視設定」を開くと、ページ全体ではなく特定範囲だけを監視したり、広告・日時・カウンターなどのノイズを除外できます。

項目内容例
監視範囲CSSセレクタで監視対象を絞ります。空欄ならページ全体です。main, article, #content
除外要素監視範囲内から無視するHTML要素を除去します。.ad, time, .sidebar
除外文字正規表現で変化ノイズを削除します。閲覧数:\s*\d+
テスト現在の設定で取得される先頭テキストを確認します。設定変更前の確認に使います。
監視範囲や除外ルールを変更すると、そのサイトのベースラインはリセットされます。次回確認で新しい基準を取得します。

タグ・ピン

タグ

行を展開し、タグ欄にカンマ区切りで入力します。タグが1件以上あるとタグフィルターバーが表示され、タグボタンや行のタグピルから絞り込めます。

ピン

後で確認したいサイトは 📍 でピン留めできます。一覧ヘッダーの 📍 または p でピン済みサイトだけを表示します。

タイル表示・画像モード

タイル表示

⊞ または t で切り替えます。カード形式で監視サイトを見渡し、クリックすると詳細オーバーレイを開きます。スマホ表示ではタイルボタンは表示されません。

画像モード

🖼 または i で切り替えます。各ページの og:image と本文内画像を抽出し、サイトごとにグリッド表示します。

画像モードでは、画像クリックでライトボックスを開き、同じサイトの前後画像へ移動できます。監視範囲が設定されている場合は、その範囲内の画像を優先して収集します。

Cloud同期

Supabaseを使うと、複数端末で同じ監視サイト一覧と確認結果を共有できます。Cloud設定がない場合は従来どおり localStorage だけで動作します。

初回設定

  1. Supabase SQL Editorで output/antenna/supabase-schema.sql を実行します。
  2. Web Antenna の ⚙ を開きます。
  3. Cloud設定 に Supabase URL、anon key、共有ID を入力します。
  4. 保存 または 同期 を押します。

同期される内容

  • URL、サイト名、タグ、ピン状態
  • 監視範囲、除外要素、除外文字
  • 最終確認日時、最終更新日時、前回取得テキスト、差分、ステータス、エラー

同期方式

サイト追加、削除、確認、タグ変更、ピン変更、監視設定変更の後に自動同期します。保存時は先に現在行を upsert し、成功後に不要行だけ削除するため、通信失敗でCloud側が空になりにくい方式です。

複数端末で同時編集した場合は、URLごとに updated_at が新しいデータを優先します。古い端末で大きく編集する前に、まず手動同期してください。

スマホ操作

スマホ幅では、一覧と詳細を切り替える表示になります。PC向けの import/export とタイル表示ボタンは表示されません。

操作動作
URLボタンURL入力欄を開閉します。入力欄はボタンを押した時だけ表示されます。
サイト行をタップ詳細画面を全画面風に開きます。
← / Esc詳細画面から一覧へ戻ります。
⚙ 設定表示テーマ、CORSプロキシ、Cloud設定を操作します。
表示テーマ設定画面からライト / ダークを切り替えます。

ホーム画面に追加

manifest.webmanifest と apple-touch-icon.png を参照しているため、対応ブラウザではホーム画面に追加した時に専用アイコンが表示されます。アイコンはA型アンテナ塔と電波アークを組み合わせたシンプルなデザインです。

import/export

PC表示では、ヘッダーの export で登録サイト一覧を antenna-sites.json として書き出し、import で復元できます。スマホ表示ではボタンを表示しません。

項目内容
exportURL、ラベル、監視設定、タグ、ピン状態を書き出します。
import既存リストに追加します。同じURLは重複登録しません。
Cloudとの関係Cloud設定中にimportすると、追加分も通常保存後に同期されます。

データ仕様

保存先キー / テーブル内容
localStorageantenna-sites監視サイト、確認結果、差分、タグ、ピン、監視設定。
localStorageantenna-themeライト / ダークテーマ。
localStorageantenna-custom-proxy任意のCORSプロキシURL。
localStorageantenna-cloud-configSupabase URL、anon key、共有ID。
Supabaseantenna_sites共有IDごとの監視サイトと確認結果。

Cloudテーブルの主キーは (profile_id, url) です。共有IDごとにデータを分離します。

トラブル対応

サイト取得に失敗する
公開CORSプロキシの制限、対象サイトのブロック、ネットワークエラーが考えられます。時間を置いて再確認するか、設定からカスタムCORSプロキシを指定してください。
関係ない小さな変化を拾う
監視範囲を main や article に絞り、広告・日時・カウンターを除外要素または除外文字で外してください。
Cloud同期に失敗する
Supabase URL、anon key、共有ID、SQLスキーマ、RLSポリシーを確認してください。ブラウザのステータスバーにHTTPエラーやテーブル名が表示される場合があります。
スマホでタイル表示が出ない
仕様です。スマホ表示ではタイルボタンを非表示にし、リストと詳細画面の切り替えに絞っています。
データが消えたように見える
タグフィルタ、ピンフィルタ、共有ID違いを確認してください。localStorageを削除した場合は、Cloud設定後に同期するか、PC表示のimportでバックアップJSONを読み込んでください。
